# Сравнение функций: YNAB‑lite PWA (MVP) vs YNAB

| Функция | YNAB‑lite PWA (этот проект, MVP) | Оригинальный YNAB |
|---|---|---|
| Учёт транзакций (доход/расход) | Да | Да |
| Несколько счетов (наличные/карта/банк) | Да | Да |
| Баланс по счетам и общий баланс | Да | Да |
| Категории (дефолтные + свои) | Да | Да |
| Архивация счетов/категорий без потери истории | Да | Да |
| Фильтры по дате/счету/категории | Да | Да |
| Поиск по заметке | Да | Да |
| Офлайн‑работа | Да (offline‑first) | Частично/зависит от клиента |
| PWA установка на iOS | Да (цель) | Нет (обычно нативные/веб) |
| Экспорт/импорт данных | Да (JSON snapshot) | Да (экспорт есть; импорт/миграции — через YNAB) |
| “Каждый рубль занят” (zero-based budgeting, назначение денег категориям) | Нет | Да |
| Месячные бюджеты/цели по категориям (targets/goals) | Нет | Да |
| Планирование/перекидывание средств между категориями (move money) | Нет | Да |
| “Age of Money” и метрики YNAB | Нет | Да |
| Отчёты/аналитика (spending, net worth и т.п.) | Нет | Да |
| Запланированные транзакции | Нет | Да |
| Повторяющиеся транзакции/автоподстановка | Нет | Да |
| Импорт из банка / автосинхронизация с банками | Нет | Часто да (по региону), плюс file import |
| Синхронизация между устройствами | Нет (планируется пост‑MVP) | Да |
| Совместный доступ/шаринг бюджета | Нет | Да (планы/шаринг в экосистеме) |
| Мультивалюта | Нет (только RUB) | Частично/ограниченно |
| Мультибюджеты (несколько бюджетов/проектов) | Нет (в MVP не заявлено) | Да |
| Аудит/история изменений/конфликты | Нет | Да (за счёт синка и инфраструктуры) |
