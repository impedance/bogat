# Промт для следующего запуска агента (организовать “оглавление в код”)

Ты — coding‑агент в этом репозитории. Твоя задача на старте — привести документацию к протоколу “документация = оглавление в код”.

## Boot sequence (обязательно)
1) Открой `README.md` и выполни инструкции из секции **«Для coding-агентов (прочитай первым)»**.
2) Прочитай `AGENTS.md` и следуй протоколу **«README как индекс (протокол)»**.
3) Выполни `rg -n "AICODE-"` и найди релевантные якоря перед любыми изменениями.
4) Прочитай `docs/context.md` и `docs/status.md` (контекст + текущий фокус).

## Основная задача (индекс README)
Если в `README.md` отсутствуют секции `Repository layout`, `Entry points`, `Search cookbook` (или они устарели относительно текущей структуры репозитория):
- создай/обнови эти секции строго по требованиям из `AGENTS.md`:
  - `Repository layout`: 10–25 ключевых путей `path/или/file` + “искать: `rg -n ...` / `AICODE-...`”
  - `Entry points`: 3–10 точек входа с путями и быстрым поиском
  - `Common tasks`: только реальные команды из `package.json`/скриптов
  - `Search cookbook`: 8–15 рабочих `rg -n ...` запросов под этот репозиторий

## Правила AICODE (обязательно)
- Используй только разрешённые префиксы из `docs/agent/ai-anchors.md`.
- Не дублируй якоря: перед добавлением всегда делай `rg -n "AICODE-"`.
- Для `AICODE-WHY/TRAP/FACT/CONTRACT/HISTORY` обязательна дата `[YYYY-MM-DD]`.
- Схему тем/namespace для текста якоря см. `docs/agent/anchor-scheme.md`.

## Завершение (обязательно)
1) Запусти проверки:
   - `npm run lint:aicode`
   - `npm test`
2) Коротко отчитай:
   - какие секции добавил/обновил в `README.md`
   - какие `AICODE-*` якоря добавил/обновил (если добавлял)
